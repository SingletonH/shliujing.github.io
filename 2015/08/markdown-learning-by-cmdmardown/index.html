Cmd Markdown的Markdown入门简记

发表于 2015-08-15   |   分类于 Tech   |     |   热度

之前学习Markdown的时候，入门的语法只花了10来分钟。但是在工具选择，以及工具的高效使用上，花了一些时间。我用过的工具大概有几款吧，回忆过来，用过的顺序依次是简书->马克飞象->Cmd Markdown->Mou->Sublime->IDEA->Atom。最好用的还是马克飞象和Sublime。

今天要介绍的就是一款非常简洁美观的Markdown工具：Cmd Markdown。一款基于Web的跨平台软件，软件的主题色是暖色的淡黄色，作者设计时还是充分考虑了文字工作者的实际应用场景。而且通过Web封装还实现了三大平台客户端的支持。相比马克飞象，没那么优美，而且在Chrome插件应用中也没有，所以我其实用的马克飞象更多。关于我对马克飞象的体验，点击这里查看。

Cmd Markdown，是跨平台，还可以云端存储分享。我当时用的时候还是免费，现在也开始收费了。这些都是独立开发者，个人完成的这么棒作品，很厉害也很不容易。为他们加油，希望他们能做的越来越好。

这里主要记录我没掌握的部分扩展语法，完整的只有一篇博文长度的官方入门教程，点击这里查看](https://www.zybuluo.com/)。扩展的快捷键我没特意去记，因为马克飞象不支持自定义快捷键，要记也是记Sublime的这套Markdown快捷键。

注意：因为是扩展的原因，很多语法博客不支持，包括LaTex公式，如果没有加载Mathjax，则是不显示的。

语法


## 生成目录
[TOC]

## 链接

### 例1：行内链接

这就是我们常用的地址：[Baidu](www.baidu.com "百度一下，你就知道" )

### 例2：参考式链接

这就是我们常用的地址：[Baidu][1]
[1]:www.baidu.com "百度一下，你就知道" 

## 引用

> 这是一级引用
>>这是二级引用
>>> 这是三级引用
>这是一级引用

## 反斜杠

使用\表示反斜杠。在你不想显示Markdown标记时可以使用反斜杠。
\*这里不会显示斜体\*


## 删除线

~~这是一条删除线~~

## 注脚

这是一个注脚测试[^footer1]。
[^footer1]: 这是一个测试，用来阐释注脚。

## 实现页内跳转

[你好](#jump)
<span id = "jump">我是一个锚点，跳到我这里来</span>
